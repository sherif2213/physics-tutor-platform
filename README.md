# منصة إدارة مركز الفيزياء

نظام إدارة متكامل لمركز الدروس الخصوصية — Next.js + Supabase + PWA offline-first.

## 1. إعداد Supabase (مرة واحدة)

1. أنشئ مشروعًا جديدًا على [supabase.com](https://supabase.com).
2. من **SQL Editor**، شغّل محتوى الملف `supabase/schema.sql` بالكامل — هذا ينشئ كل الجداول، الفهارس، وسياسات الأمان (RLS).
3. من **Authentication → Users**، أنشئ مستخدمًا واحدًا (بريد إلكتروني + كلمة مرور) — هذا هو حساب دخول المُدرّس. لا حاجة لتسجيل دخول الطلاب.
4. من **Project Settings → API**، انسخ:
   - `Project URL`
   - `anon public key`

## 2. الإعداد المحلي

```bash
npm install
cp .env.local.example .env.local
# افتح .env.local وضع القيم من الخطوة السابقة
npm run dev
```

افتح `http://localhost:3000` وسجّل الدخول بالحساب الذي أنشأته في Supabase.

## 3. النشر (Deployment)

الطريقة الأسهل: [Vercel](https://vercel.com) — اربط المستودع وأضف نفس متغيرات البيئة (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`) من إعدادات المشروع على Vercel.

بعد النشر، افتح الرابط من هاتفك واختر "إضافة إلى الشاشة الرئيسية" (Android) أو "إضافة إلى الشاشة الرئيسية" من قائمة المشاركة في Safari (iPhone) لتثبيته كتطبيق.

## البنية

- `app/` — صفحات Next.js (App Router): تسجيل الدخول، الرئيسية، الطلاب، الحضور، المصروفات، المجموعات، التقارير، الإعدادات.
- `lib/dataStore.js` — طبقة موحّدة للقراءة والكتابة: كل صفحة تتعامل معها فقط، وهي المسؤولة عن العمل أونلاين/أوفلاين.
- `lib/offlineDb.js` — تخزين IndexedDB (عبر مكتبة `idb`) لتخزين البيانات محليًا وقائمة انتظار التغييرات غير المُزامنة.
- `lib/syncEngine.js` — يعيد تشغيل التغييرات المخزنة محليًا على Supabase بمجرد عودة الاتصال، بأمان (idempotent) عبر جدول `sync_log`.
- `supabase/schema.sql` — مخطط قاعدة البيانات الكامل.

## آلية العمل بدون إنترنت

- كل قراءة تُخدَّم فورًا من IndexedDB، ثم تُحدَّث في الخلفية من Supabase عند توفر الاتصال.
- كل كتابة (حضور، دفع، بيانات طالب) تُطبَّق فورًا محليًا ثم تُضاف لقائمة انتظار المزامنة.
- عند عودة الاتصال (أو كل 30 ثانية أثناء الاتصال)، تُعاد كل العمليات المعلّقة على Supabase تلقائيًا.
- كل عملية لها معرّف فريد من الجهاز، مسجّل في جدول `sync_log` على الخادم، بحيث لا تُطبَّق أي عملية مرتين ولا تُفقد أي بيانات حتى لو انقطع الاتصال أثناء المزامنة.
- حذف الطلاب هو حذف ناعم (soft delete) — سجلات الحضور والمدفوعات تبقى محفوظة دائمًا.
- التطبيق مُسجَّل كـ PWA (service worker + manifest) بحيث يفتح حتى بدون إنترنت ويمكن تثبيته على الهاتف كتطبيق أصلي.

## ملاحظات حول ما يحتاج تخصيص إضافي

- أيقونات PWA في `public/icons/` هي أيقونات مبدئية بسيطة — استبدلها بشعار السنتر الفعلي (المقاسات: 192×192، 512×512، ونسخة maskable 512×512).
- تصدير PDF/Excel في صفحة التقارير يعمل فعليًا (jsPDF + SheetJS) ويغطي التقارير الأساسية المطلوبة؛ يمكن توسيعه بمزيد من التنسيقات حسب الحاجة.
- "الحاضرون/الغائبون اليوم" في لوحة التحكم متروكة لك لتحديد آلية الربط بين "اليوم" ورقم الحصة، لأن الحصص مرقّمة (1-12) وليست مرتبطة بتاريخ محدد في المواصفات الأصلية — يمكن إضافتها بسهولة إذا حددت لي كيف تريد الربط (مثلاً: كل حصة لها يوم أسبوعي ثابت).
